## 1. üî¥ Verified critical bug ‚Äî fix first

### `status-dot-active-prop-ignored` ‚Äî live health dots render gray "unknown" on Overview, Nodes, and Map

**This is the single highest-value fix in the entire review.** The core "health at a glance" signal is dead on the three most important fleet surfaces.

`StatusDot.vue` declares only these props: `status`, `online`, `pulse`, `label`, `class` (verified, `StatusDot.vue:7-19`). Its resolution logic: if `online !== undefined` ‚Üí online/offline, **else** fall back to `status` (default `"unknown"` ‚Üí gray dot), and the pulse only animates when `resolved === "online"`.

But the hottest call sites pass props that **don't exist**:

| Call site | Code | Result |
|---|---|---|
| `OverviewView.vue:212` | `<StatusDot :active="node.online" :pulse="node.online" />` | `active` ignored ‚Üí `online` undefined ‚Üí **gray** |
| `OverviewView.vue:319` | `<StatusDot tone="warning" pulse />` | `tone` ignored ‚Üí **gray**, never amber |
| `NodesView.vue:333` | `<StatusDot :active="node.online && !node.disabled" ... />` | **gray** on the dedicated Nodes page |
| `MapView.vue:504` | `<StatusDot :active="node.online" ... />` | **gray** on the map |

Only `WireGuardView.vue:200` and `InventoryView.vue:422` use the correct `:online=` form and actually work.

**Fix (S, ~30 min):**
1. Change the four call sites to `:online="..."` (boolean cases) and add a real `tone?: 'success'|'warning'|'info'|'destructive'` (or `status`) for the Overview approvals dot at `:319`.
2. Add a `status?` path so degraded/pending can eventually be expressed.
3. **Prevent recurrence:** Vue silently drops undeclared attributes onto the root element ‚Äî that's why this slipped past `vue-tsc`. Pair this fix with **`vis-centralize-status-tone-helper`** (P0 below) so all status color flows through one typed helper. Consider an explicit prop allowlist / `inheritAttrs: false` on StatusDot so stray attrs surface.

Reference shape: NodeLite `NodeCard` status pill and nezha `transfer.tsx:32-41` (`statusMeta ‚Üí Badge variant`) ‚Äî one source of truth, always bound to real state.

## 2. What lattice already does well (keep these)

The review gave lattice real credit ‚Äî on several axes it **beats all three references**:

- **Theming is best-in-class.** Complete OKLCH token set on shadcn-vue New-York base, re-skinned slate-indigo, with **first-class status tokens** (`--success/--warning/--info/--destructive` + `-foreground` pairs). Two-axis theming (light/dark FOUC-free via `public/theme-init.js` + 10 brand palettes/custom hex with auto-derived contrast foreground). nezha declares `chart-1..5` and never uses them; NodeGet/NodeLite use hsl/hex. (`app.css:36-159`, `theme/palettes.ts`)
- **States are a real state machine.** `DataState.vue` centralizes loading‚Üískeleton / error‚Üícard-with-`request_id`+Retry / empty‚Üí`EmptyState` / default. Coverage is stronger than all three references. (The fix in ¬ß5 is to *use* its last-good retention.)
- **Security-grade action model.** plan ‚Üí approve ‚Üí apply with client-side sha256 binding; secret-free `has_*` API; no fire-and-forget mutations. None of the references match this discipline.
- **CSP-safe, dependency-light charting** via inline-SVG `TrendChart` ‚Äî the right posture (NodeLite proves a full dashboard with zero chart libs).
- **Declarative single-source IA** ‚Äî one `NAV[]` array drives router + sidebar, scope-filtered by `auth.canAny`. Clean.
- **Full en/zh-CN i18n** infrastructure with namespaced catalogs.
- **Polling discipline & a11y groundwork** (skip link, landmarks, aria-current from the prior sweep).

The theme are not "lattice is weak" ‚Äî it's "lattice has an excellent foundation with a few sharp gaps and a lot of copy-paste that should be consolidated."

## 4. Consensus findings (surfaced by multiple independent reviewers = highest confidence)

These came up in 2‚Äì4 separate dimensions independently ‚Äî strong signal:

- **Native `window.confirm` for destructive actions** (Inventory, Monitoring) ‚Äî flagged **4√ó** (Tables, Fleet, Forms, States). ‚Üí one shared themed `ConfirmDialog`.
- **No "Live / Updated Ns ago" freshness label** outside Overview ‚Äî flagged **3√ó** (Fleet, Forms, States). Critical for a socket-less polling UI. ‚Üí one `useLiveLabel` composable.
- **Shared `DataTable` / status SSOT / per-node sparklines / node deep-link / sidebar-persist** ‚Äî each flagged **2√ó**.

The duplication in the findings *is the finding*: lattice's biggest structural debt is **copy-paste across views** (confirm dialogs √ó13, plan-review dialogs √ó9, status-color mapping √ó12, table scaffolding √ó4). Consolidating these is what makes every other improvement cheap.
