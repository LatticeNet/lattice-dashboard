# Lattice Dashboard — UX + Code Audit & Redesign Roadmap (2026-06-26)

Owner critique (verbatim themes): color scheme & interaction logic feel unsatisfying;
overuse of **non-interactive "card"** widgets that only display and never drill through;
Overview "Fleet 15/15" tile is a dead end; Node cards expose a terminal but you can't
**open the node itself**; the **Groups** tab is confusing because grouping isn't reflected
on Nodes and you can't assign group/leader at registration; the **Map** shows the internal
Docker IP `172.18.0.1` instead of the public IP; **Monitoring** doesn't scale to many
monitors/targets/nodes and the log section stacks endlessly; **Tasks** layout + history are
poor; **Audit** search/guidance is weak (design for a geek user); and the **Networking /
Inbounds / Profiles / Subscriptions** config surfaces should offer a **review-before-apply
preview** showing what the server and each node will change.

Method: 9 parallel agents each read the real Vue/Go/agent code for one surface and tied
findings to the owner's complaints (file:line anchored), then a synthesis pass produced the
roadmap below. Source tree: `/Users/cdcd/roobli/RTFS_justTaste/Probe-Dashboards/Lattice`.

## 0. Two concrete defects found (correctness, not polish)

### D1 — Map shows the Docker bridge IP as the node's "public IP" (root-caused)
- The agent reports an **empty** `public_ip` by default (`lattice-node-agent/cmd/lattice-agent/main.go:139,217-226,455-462`).
- The server then fills it by **observing the connection source**: `resolvePublicIPs` → `clientIP(r)` (`lattice-server/internal/server/server.go:3031-3043, 4702-4721`).
- The server runs **inside Docker behind nginx**, so the observed source is the **bridge gateway `172.18.0.1`**. `resolvePublicIPs` persists it with **zero routability validation** (the existing `validateReportedPublicIP` at `server.go:4495` is applied only to the *agent-reported* value, never to the observed one), so the bad IP reaches `node.PublicIP` (`server.go:4083/4127`), the Map geocoder, and DDNS publish. Geo-lookup then correctly rejects it: *"ip is not a routable public address."*
- Even with `TrustProxy` correct, "observe the source IP" is structurally wrong for an agent co-located behind a CDN/proxy. **The right fix is agent-side active public-IP discovery** (resolver list / static / custom script), with the server guarding against ever persisting a private/bridge IP.

### D2 — Phantom metrics fields → node speeds likely always `0 B/s`
- TS `Metrics` declares `net_rx_speed`/`net_tx_speed`/`load5`/`load15` (`types.ts:42-51`) but the Go `Metrics` struct emits none of them (`lattice-sdk/model/model.go:155-166`, only `NetRxBytes`/`NetTxBytes`).
- Consumed at `NodeCard.vue:300-307` and the header bandwidth total (`fleet.ts:388`) → these almost certainly render `0 B/s` permanently. Verify against a live payload; then either add the fields to the Go struct + agent collector, or drop them from TS.

## 1. Themes

1. **Interaction & drill-through** (the dominant complaint). KPI `StatCard`s, `DataTable`
   rows, Overview `NodeCard`s, Map markers, and faux-hover list rows all *display* but never
   *navigate*. No systemic "this surface drills in" convention — interactivity is opt-in per
   view with three competing hover idioms; deep-link targets (node, status filter) are only
   half-wired. Fixing the shared primitives repairs Overview/Monitoring/Tasks/Audit/Map at once.
2. **IP / geo / DDNS correctness** (D1). Plus: no internal/LAN IP captured anywhere; DDNS has
   no custom-script provider; failures surface as silent no-ops.
3. **Config safety / dry-run** ("让我心里有底"). A mature plan→approve→apply backend already
   exists (SHA-pinned, TOCTOU-guarded), but plans render the **full new artifact, not a
   before/after diff**; server-vs-node split is only freeform markdown in selfdns; shared-entity
   edits (Inbounds/proxy-users/DNS) **apply blind** with no fan-out/blast-radius preview;
   GeoRouting preview is a dead-end with no apply path; payload shapes are inconsistent.
4. **Scale & density.** Monitoring: flat non-virtualized list, unbounded all-node result fetch
   every 8s, one blended heat strip, interleaved 40-row log. Tasks: unbounded on-disk retention,
   no rerun/cancel/delete/pagination, full stdout dumped inline, script body never returned.
   Audit: capped at 500 with offset hardcoded to 0 (silent data loss), exact-match-only search,
   no time range, no export.
5. **Visual identity.** Clean OKLCH dual-axis theming but reads as a generic shadcn-vue reskin
   (barely-tinted slate neutrals, stock-indigo accent, default chart ramp). Actionable vs.
   decorative surfaces are visually indistinguishable; `surface-interactive` press feedback is
   inert (`app.css:355-357`); audit decision colors misuse node-health green/red/gray so
   `observe` reads as "no data."

## 2. Quick wins (high impact, S effort)

- **Guard the observed IP** — run the `clientIP` candidate through `validateReportedPublicIP`
  before adopting it in `resolvePublicIPs`; keep last-known on failure; log it. Stops
  `172.18.0.1` ever reaching Map/DDNS. (`server.go:3031`)
- **`StatCard` gets an optional `to`** (RouterLink + cursor + focus ring + corner chevron); wire
  Overview KPIs: online → `nodes?status=online`, approvals → approvals, queued → `tasks?status=queued`.
- **Seed `NodesView`/`TasksView` filters from `route.query`** (status/region) so KPI deep-links land
  pre-filtered (mirror the node deep-link at `NodesView.vue:225/231`).
- **Make the whole NodeCard clickable** → open node detail (drop name-only button; footer buttons `@click.stop`).
- **Overview NodeCards open the node**: drop `:selectable="false"` (`OverviewView.vue:300`), wire `@select`.
- **Give `surface-interactive` a real hover lift + true press settle** and make it the single canonical token.
- **Fix audit decision colors**: allow=green, deny=destructive, observe=blue/amber informational.
- **Group chips on NodeCard + node detail** (data already on the wire as `node.group_ids`).

## 3. Recommended sequence

> Ordering principle: correctness first, then the foundational interaction primitives + the
> drill-in target (node detail) + group coherence (knocks out most complaints), then the marquee
> config dry-run, then scale/density, then the taste-dependent palette last.

1. **IP correctness** (S, server-mostly). Guard observed IPs + keep last-known + gate observation
   behind an explicit option; replace MapView's generic toast with a per-node "no routable public
   IP — configure IP mode" badge. *Owner's stated top priority.*
2. **Shared interaction primitives + Overview** (M). `StatCard.to`, `DataTable.rowTo`, base `Card`
   `interactive`, query-param filter seeding, the visible affordance signifier (chevron/lift/focus
   ring). Wire Overview KPIs/NodeCards/approval+activity rows/"View all". One convention, five surfaces.
3. **Node Detail page** (M) — `NodeDetailView` at `/nodes/:id`. Sections: live status+metrics,
   network/IP config (public+internal IP + provenance), group membership, DDNS bindings, agent
   version/update, recent audit-for-node, embedded terminal, danger actions. Most data is already
   on the wire; needs the route + (optional) single-node GET + rename/delete handlers.
4. **Groups coherence** (S→M). Surface group chips + "manage groups" on Nodes (reuse existing
   `/api/groups/members`); assign group(s) at enroll (small server delta); make **leader** a real
   `Group.LeaderID` field (the only net-new field — retires the regex heuristic in `PolicyView`).
5. **Agent-side public-IP discovery + internal/LAN IP + per-node IP config** (L). Resolver list /
   static / script (nezha-style); `InternalIP[v6]` on the model + agent enumeration; `NodeIPConfig`
   pushed via the existing agent-config channel. The structural fix the slice-1 guard protects.
6. **Custom-script DDNS provider** (L). `DDNSProviderScript` that runs **on the target node via the
   existing taskexec sandbox** (not the control plane), receiving `DDNS_IP/DDNS_DOMAIN/DDNS_TYPE`,
   returning the IP on stdout.
7. **Dry-run upgrade** (L). Each renderer also emits the previous artifact + unified diff;
   `PlanReviewDialog` renders red/green deltas + a **Server tab + per-node tabs**. One `PlanEnvelope`
   shape (`server_changes[]` + `targets[]`) retires the GeoRouting special-case and collapses the
   duplicated plan-state across 8 views. Make **Preview the primary CTA**.
8. **Fan-out blast-radius preview** (M). Generalize `handleGroupPolicyPlan`'s per-node aggregation
   into a reusable impact endpoint so Inbounds/Subscriptions/DNS edits show "affects N nodes, here is
   each new plan" before writing; give GeoRouting a real apply path.
9. **Scale & density pass** (L, 3 parallel lanes). Monitoring: search/filter/group toolbar +
   virtualized list (`useVirtualList` already a dep), bounded results fetch (limit+node+since),
   per-node small-multiples heat, pausable/tailable virtualized log. Tasks: active-vs-history split,
   return script body + detail drawer, rerun/cancel/delete, pagination/filters, retention caps. Audit:
   `key:value` token query bar + free-text, time-range, faceted clickable chips, offset pagination,
   NDJSON/CSV export.
10. **Visual identity** (M, taste-dependent, last). Deepen dark surfaces toward near-black cool
    slate; replace stock-indigo with a signature accent + distinct data-accent (cyan/teal); re-space
    the chart ramp. Propose 2–3 options in the live AppearanceView preview before touching tokens.

## 4. Open questions for the owner (decisions that branch the work)

1. **IP fix posture**: is **agent-side discovery the single source of truth** (recommended), with
   the server guard as a safety net? Or should we *also* fix server-side `TrustProxy`/CF-Connecting-IP
   (nginx realip) as a fallback?
2. **Script execution boundary**: confirm DDNS / IP-mode scripts run **on the target node** via the
   taskexec sandbox (interpreter allowlist, rlimit, `-allow-exec`), not in the control plane.
3. **GeoRouting**: wire its preview into the standard `Approval{Plugin:"georouting"}` pipeline so it's
   approvable/appliable (recommended), or keep it advisory and just label it as such?
4. **Preview mandatory?** Should "Preview changes" become a required confirmation step before every
   apply, or stay an opt-in CTA?
5. **Node Detail MVP scope** + whether it's a full route or a master-detail split on wide screens.
6. **Retention caps** for Tasks/results and the audit window (current on-disk growth is unbounded).
7. **Audit search depth**: full `key:value` token language + server `q` substring/prefix param
   (geek-first, larger), or start with just the missing facets (actor/scope/token) + time range?
8. **Palette direction**: pick among 2–3 proposed terminal/control-plane identities + the signature
   accent + secondary data-accent hues before any global token change.

## 5. Per-surface detail (condensed; full evidence in session transcript)

### Design system / color / cards
OKLCH light/dark + 9 runtime brand palettes (`app.css:67-159`, `palettes.ts:51-97`). Base `Card`
is a static div; `StatCard`/`NodeCard` build on it. `surface-interactive` exists but is wired into
only 4 views; its `:active` rule is inert. Fix systemically: `interactive` on base Card, `to` on
StatCard, `rowTo` on DataTable, one canonical hover/press token, a visible affordance signifier.

### Overview
Five display-only blocks; only Refresh + retry are clickable. KPI tiles, NodeCards
(`:selectable="false"`), approval/activity rows (fake hover affordance), and Fleet-Health/region
tiles all dead-end. Add `to` to StatCard, wire KPIs to filtered destinations, make NodeCards open
the node, give rows real navigations + "View all" header links, seed filters from query.

### Nodes + node detail
Only the node **name** opens a cramped modal; the card body is dead despite hover styling; no
`/nodes/:id` route. Modal shows a fraction of available data; `group_ids` is on the wire but never
rendered. Promote to a real `NodeDetailView` page; make the whole card navigate; embed the terminal
(extract `NodeTerminal` from `TerminalView`). Mostly dashboard-only; optional new server handlers
(single-node GET, rename/delete) + the D2 metrics-field decision.

### Groups + enrollment
Groups are a **genuine first-class entity** server-side (`model.go:422-436`; full CRUD +
`/api/groups/members`, `preview`, `seed`, `reorder`), but: membership is **invisible on Nodes** by
default (NodeCard has no group chip; default groupBy is region); **enrollment can't assign a group**
(`server.go:1604-1610`); **"leader" is a fiction** (regex on role names in `PolicyView.vue:512-514`,
no `Group.LeaderID`). Fix: group chips + manage-groups on Nodes (reuse existing endpoint, S),
assign-at-enroll (small server delta), real `LeaderID` field (only net-new field), cross-link
Nodes↔Groups↔Policy.

### Map + IP + DDNS
See D1. Markers only populate an inline lat/lon editor — no drill-through. Recommendations: stop
persisting non-routable observed IPs; agent-side discovery; capture internal IP; per-node IP config;
custom-script DDNS; drill markers to node detail; surface resolution failures as per-node badges.

### Monitoring
Flat non-virtualized monitor list; results endpoint returns **unbounded all-node history** polled
every 8s and fully re-sorted each tick; one blended heat strip; interleaved 40-row log that "stacks."
Fix: search/filter/group toolbar + virtualize; bound the fetch (limit+node+since, server clamps like
logstore); per-node small-multiples heat; pausable/tailable virtualized log; per-monitor drill-in
route; density toggle; move create-form into a dialog.

### Tasks
Script body never returned to UI (only SHA+size); no rerun/cancel/delete; **unbounded retention**
(contrast `maxSessions`/`maxMonitorResults` which DO cap); no active-vs-history split; full stdout
dumped inline; no pagination/filter. Fix: return script + detail drawer; rerun/cancel/delete
endpoints; active-vs-history layout; stat cards filter on click; server pagination + retention caps;
humanized status + duration + per-node progress.

### Audit
Exact-match-only action filter (no prefix/substring); server supports actor/token/scope filters the
UI never exposes; **no time-range** anywhere; offset hardcoded to 0 → results beyond 500 unreachable
(silent data loss); three display-only cards; DataTable search/sort disabled; every row force-renders
a `<pre>`; filters not in URL; no export; `observe` reads as gray "unknown." Fix: `key:value` token +
free-text query bar (server `q`), time-range, faceted clickable chips, dense expandable rows + drawer,
URL sync + saved queries, offset pagination, NDJSON/CSV export, decision color fix, teaching empty state.

### Config dry-run (cross-cutting)
A mature plan→approve→apply pipeline exists (SHA-pinned, TOCTOU-guarded, `PlanReviewDialog` +
`usePlanDigest`). Gaps: plans show full NEW config not a **diff**; server-vs-node split only in
selfdns markdown; shared-entity edits apply blind (no fan-out); GeoRouting preview can't be applied;
inconsistent payload shapes. Fix: before/after diff in every renderer; one `PlanEnvelope`
(`server_changes[]` + `targets[]`); fan-out impact endpoint (generalize group-policy aggregation);
GeoRouting apply path; Preview as primary CTA.
